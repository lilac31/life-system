# 多设备同步使用指南

## 📱 快速开始

### 第一台设备（主设备）

1. **获取 API Key**
   - 访问 [JSONBin.io](https://jsonbin.io) 免费注册
   - 创建 Master API Key

2. **配置同步**
   - 打开应用的"同步设置"
   - 输入 API Key 并验证
   - ✅ 配置完成！使用应用时数据会自动上传云端

3. **保存 Bin ID**
   - 创建或编辑任何数据后，系统会自动创建云端存储
   - 在"同步设置"中查看并**复制完整的 Bin ID**
   - 格式：24位字母数字组合，如 `6a7b8c9d0e1f2a3b4c5d6e7f`

---

### 第二台设备（其他设备）

1. **输入相同的 API Key**
   - 打开应用的"同步设置"
   - 输入与主设备相同的 API Key
   - 验证成功

2. **连接到已有数据**
   - 点击"使用已有数据"按钮
   - 粘贴主设备的 **Bin ID**
   - 点击"保存 Bin ID"
   - 刷新页面

3. **✅ 同步完成！**
   - 所有设备的数据现在会自动同步

---

## 🔄 同步机制

- **自动上传**：每次保存数据时自动上传到云端
- **自动下载**：每30秒检查一次云端更新
- **智能合并**：云端数据优先，自动合并冲突

---

## ❓ 常见问题

### 为什么不能自动查找 Bin ID？

JSONBin API 的 Bin 列表返回格式不稳定，自动查找容易失败。手动复制 Bin ID 是最可靠的方式。

### 如果 Bin ID 丢失了怎么办？

1. 在任一已同步的设备上打开"同步设置"
2. 复制显示的 Bin ID
3. 在新设备上按照"第二台设备"的步骤操作

### 可以同时使用多个 Bin 吗？

不建议。一个 API Key 应该只使用一个 Bin ID，确保所有设备数据一致。

---

## 🛡️ 安全提示

- **API Key** 是你的私密凭证，不要分享给他人
- **Bin ID** 可以在你的设备间分享，但也不应公开
- 所有数据都加密传输到 JSONBin 云端

---

## 📞 技术支持

如遇问题，请检查：
1. 网络连接是否正常
2. API Key 是否有效
3. Bin ID 格式是否正确（24位字母数字）
4. 浏览器控制台是否有错误信息
