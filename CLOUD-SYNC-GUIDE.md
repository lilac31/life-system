# 云端同步设置指南

## 🌟 推荐方案：GitHub Gist（免费、稳定）

GitHub Gist是GitHub提供的代码片段分享服务，免费且稳定，非常适合作为数据同步的云存储。

### 🔧 设置步骤

1. **注册GitHub账户**：
   - 访问 [GitHub.com](https://github.com) 并注册免费账户
   - 如果已有账户，直接登录即可

2. **创建访问令牌**：
   - 访问 [GitHub令牌设置](https://github.com/settings/tokens)
   - 点击"Generate new token (classic)"
   - 输入令牌名称（如：生活管理系统）
   - 选择过期时间（推荐：90天）
   - 勾选"gist"权限
   - 点击生成令牌
   - 复制生成的令牌（以"ghp_"开头的字符串）

3. **在应用中设置**：
   - 在应用的"数据同步设置"窗口选择"GitHub Gist"
   - 粘贴访问令牌
   - 点击"保存令牌"

### ✅ 优势

- **完全免费**：GitHub免费账户包含Gist功能
- **稳定可靠**：由GitHub提供，长期稳定
- **隐私保护**：可设置为私有Gist，只有你能访问
- **无限制**：单个Gist可存储数MB数据，远超需求
- **随时访问**：即使更换应用，也能直接从GitHub获取数据

## 🔄 同步流程

1. **初始设置**：在第一个设备上设置GitHub令牌
2. **数据上传**：修改内容后自动上传到GitHub Gist
3. **其他设备**：在其他设备上使用相同令牌即可同步数据
4. **双向同步**：任一设备修改，其他设备刷新后即可获取

## 🔐 数据安全

- **加密传输**：所有数据通过HTTPS加密传输
- **私密存储**：数据存储在私有Gist中，只有你能访问
- **随时控制**：可在GitHub随时删除或修改Gist
- **本地备份**：数据同时保存在本地和云端

## 💡 其他选项

### JSONBin.io（不推荐）
- 免费，但不稳定
- 可能有数据丢失风险
- 适合测试，不适合重要数据

### 自建服务器（高级用户）
- 完全控制数据
- 需要技术能力
- 适合企业和团队使用
