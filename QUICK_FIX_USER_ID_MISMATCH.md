# 用户ID不匹配 - 快速修复指南

## 🚨 错误提示

```
云端数据的用户ID与当前用户ID不匹配
```

---

## ⚡ 快速解决（3步）

### 方法1️⃣：使用正确的用户ID（推荐）

**适合**：新设备同步旧数据

```
1. 打开旧设备 → 点击 👤 用户 → 复制用户ID
2. 打开新设备 → 点击 👤 用户 → 粘贴用户ID → 保存
3. 等待自动同步 ✅
```

---

### 方法2️⃣：覆盖云端数据

**适合**：不需要云端数据，重新开始

```
1. 输入新的用户ID → 保存
2. 出现错误提示 → 点击"确定"
3. 云端数据被清除，本地数据上传 ✅
```

⚠️ **警告**：云端旧数据会被永久删除！

---

## 🔍 我该用哪个方法？

| 情况 | 使用方法 |
|------|---------|
| 我有旧设备，想同步数据到新设备 | **方法1** |
| 我没有旧设备，也不需要云端数据 | **方法2** |
| 我想保留旧数据 | 先导出备份，再用方法2 |

---

## 📝 详细步骤

### 方法1详细步骤

**在旧设备上：**
```
1. 打开应用
2. 点击右上角 👤 用户
3. 看到完整的用户ID（如 user_1700123_abc）
4. 点击"复制"按钮
5. 通过微信/邮件发送给自己
```

**在新设备上：**
```
1. 打开应用
2. 配置JSONBin API Key（与旧设备相同）
3. 点击 👤 用户
4. 点击"修改用户ID"
5. 粘贴刚才复制的用户ID
6. 点击"保存"
7. 等待3秒，页面自动刷新
8. 完成！数据已同步 ✅
```

### 方法2详细步骤

```
1. 点击 👤 用户
2. 点击"修改用户ID"
3. 输入新的用户ID（或点击"生成新ID"）
4. 点击"保存"
5. 看到错误提示"用户ID不匹配"
6. 阅读弹窗说明
7. 点击"确定"（确认覆盖云端数据）
8. 等待上传完成
9. 完成！使用新用户ID ✅
```

---

## ⚠️ 注意事项

### ✅ 方法1的优点
- 保留所有云端数据
- 多设备完美同步
- 无数据丢失风险

### ❌ 方法2的风险
- 云端旧数据会被删除
- 无法恢复（除非有备份）
- 旧用户ID失效

---

## 🆘 还是不行？

### 检查清单

- [ ] 网络连接正常
- [ ] JSONBin API Key已正确配置
- [ ] 用户ID完全一致（无空格、大小写正确）
- [ ] 使用的是同一个JSONBin账户

### 终极方案

如果以上都不行：

1. **导出当前数据备份**
   ```
   点击 📦 数据 → 导出数据 → 保存JSON文件
   ```

2. **清除所有设置**
   ```
   浏览器控制台(F12)运行：
   localStorage.clear()
   location.reload()
   ```

3. **重新配置**
   ```
   配置API Key → 生成新用户ID → 导入备份数据
   ```
